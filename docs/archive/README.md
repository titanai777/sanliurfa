# Archive Index

Use archive folders for historical records, not for active delivery decisions.

## Paths
- `docs/archive/cleanup/`: dated cleanup, dirty-worktree, and verification records
- `docs/archive/phases/`: legacy phase and summary references

## Rule
- If a file exists only to explain a past cleanup pass or dated verification run, archive it.
- Active operational policy stays in root or top-level `docs/`.
